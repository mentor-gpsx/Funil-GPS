# Story 1.1: Fix Funil de Vendas Data Sync: 98→2 leads loss (P0 critical path)

**Story ID:** 1.1  
**Epic:** Data Integrity & Synchronization  
**Priority:** P0 CRITICAL  
**Severity:** Blocks production shipping  
**Date Created:** 2026-04-23  
**Status:** Ready for Review  

---

## Story Summary

The Funil de Vendas dashboard reports **2 leads in Prospecção** when **98 should display**. Root cause is a **compounding chain of 5 P0 defects** in the ClickUp fetch pipeline:

1. Frontend hardcoded username filter drops 96 leads
2. No ClickUp pagination (truncates at 100)
3. `include_closed=false` by default (excludes completed deals)
4. Silent mock fallback on error (HTTP 200 with fake data)
5. Missing `.env` loading (API key always empty locally)

**Acceptance Criteria:** All 12 defects identified in DIAGNOSTICO_FUNIL_DEFECTS.md must be resolved. **Definition of Done:** `/api/funil-by-user` count within ±5% variance of ClickUp UI.
