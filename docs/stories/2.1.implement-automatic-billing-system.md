## Dev Notes

### Architecture Context
Detailed architecture defined in `FINANCIAL_SYSTEM_DESIGN.md` (Etapa 5):

**Payment Flow:**
```
Customer signs up → Chooses payment method
├─ PIX Automático: Authorizes once, then automatic debits
├─ Boleto: Gets link, pays manually (fallback)
└─ Credit Card: High fees, last resort

Daily (00:05): Charge orchestrator runs
├─ Finds subscriptions with next_charge_date = today
├─ Generates charge record
├─ Calls Stripe/Gerencianet API
└─ Logs result

Webhook arrives (customer pays):
├─ Update charge.status = "paid"
├─ Schedule next charge (current_period_end + 1 day)
└─ Emit charge.paid event → Dashboard refreshes
```

**Key Design Decisions:**
1. **Stripe as v1 gateway** - Quick, reliable, supports both PIX + Boleto
2. **Email-based customer matching** - Deduplication key (normalized, lowercase, trimmed)
3. **Subscription-centric model** - Charges always belong to subscription
4. **Event-driven architecture** - System responds to payment events, not queries
5. **Forecast with confidence scores** - PIX=95%, Boleto=60%, CC=40%

## Database Tables (Reference)

```sql
-- Customers
CREATE TABLE customers (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  phone TEXT,
  payment_method TEXT CHECK (payment_method IN ('pix', 'boleto', 'cc')),
  pix_authorized BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Plans
CREATE TABLE plans (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  amount_cents INTEGER NOT NULL,
  interval TEXT CHECK (interval IN ('monthly', 'annual')),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Subscriptions
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY,
  customer_id UUID REFERENCES customers(id),
  plan_id UUID REFERENCES plans(id),
  status TEXT CHECK (status IN ('active', 'past_due', 'suspended', 'canceled')),
  started_at TIMESTAMP DEFAULT NOW(),
  next_charge_date DATE NOT NULL,
  canceled_at TIMESTAMP NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Charges
CREATE TABLE charges (
  id UUID PRIMARY KEY,
  subscription_id UUID REFERENCES subscriptions(id),
  customer_id UUID REFERENCES customers(id),
  amount_cents INTEGER NOT NULL,
  status TEXT CHECK (status IN ('pending', 'paid', 'failed', 'expired')),
  payment_method TEXT,
  due_date DATE NOT NULL,
  paid_at TIMESTAMP NULL,
  failed_count INTEGER DEFAULT 0,
  next_retry_at TIMESTAMP NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  gateway_charge_id TEXT
);

-- Webhooks
CREATE TABLE webhooks (
  id UUID PRIMARY KEY,
  event TEXT NOT NULL,
  charge_id UUID REFERENCES charges(id),
  payload JSONB,
  processed_at TIMESTAMP,
  received_at TIMESTAMP DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_subscriptions_next_charge ON subscriptions(next_charge_date);
CREATE INDEX idx_charges_due_date ON charges(due_date);
CREATE INDEX idx_charges_status ON charges(status);
CREATE INDEX idx_webhooks_event ON webhooks(event);
```
