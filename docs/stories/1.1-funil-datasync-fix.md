## Implementation Plan (Phase 1 — P0 Critical Path)

### F1: Frontend Username Bucket Filter
**File:** `funil.html:4381-4390`  
**Effort:** 30 min

**Current (broken):**
```javascript
['maria', 'nicolas', 'kennyd', 'gabriel'].forEach(userKey => {
  if (userKey === 'maria') userData = usuarios.find(u => u.nome.toLowerCase().includes('maria'));
  // ... only renders if username contains hardcoded substring
});
```

**Problem:** If ClickUp assignee username doesn't contain `maria|nicolas|kennyd|gabriel`, accordion never renders.

**Fix:** Replace with dynamic mapping from `CONFIG.SELLERS` (lines 1680-1685) or render accordions directly from `data.usuarios` array.

---

### F2: ClickUp Pagination
**File:** `api/clickup.js:30, 139-180`  
**Effort:** 45 min

**Current (broken):**
```javascript
path: `/api/v2/list/${listId}/task?include_subtasks=false&limit=500&...`,
const response = await fetchClickUpList(CRM_VENDAS_LIST_ID);  // Single call, no loop
```

**Problem:** ClickUp hard-caps at 100 tasks per page. If list > 100 tasks, everything past #100 is lost.

**Fix:** Add pagination loop in `fetchClickUpList()` and `loadFunilByUser()`:
```javascript
let allTasks = [];
for (let page = 0; ; page++) {
  const tasks = await makeRequest(listId, page);
  allTasks.push(...tasks);
  if (tasks.length < 100) break;
}
return { tasks: allTasks };
```

---

### F3: Include Closed Tasks
**File:** `api/clickup.js:30, 158`  
**Effort:** 15 min

**Current (broken):**
```javascript
path: `/api/v2/list/${listId}/task?include_subtasks=false&limit=500&...`
```

**Problem:** ClickUp defaults to `include_closed=false`, excluding "Pago" (completed deals) stage.

**Fix:** Append `&archived=false&include_closed=true` to URL.

---

### F4: Mock Fallback Error Handling
**File:** `api/clickup.js:143, 204-207`  
**Effort:** 20 min

**Current (broken):**
```javascript
} catch (error) {
  return getMockFunilByUser();  // Silent fallback, HTTP 200, fake data
}
```

**Problem:** Any API error → returns 2-user mock with zero visibility of failure.

**Fix:** Return HTTP 500 with error details:
```javascript
} catch (error) {
  console.error('[ClickUp] Sync failed:', error.message);
  res.writeHead(500);
  return res.end(JSON.stringify({
    error: 'ClickUp sync failed',
    message: error.message,
    source: 'error'
  }));
}
```

## Technical Details

### Root Cause Analysis
- **Primary:** Frontend `.includes()` match on hardcoded 4 users drops 96+ leads
- **Secondary:** ClickUp pagination truncates at 100 tasks
- **Tertiary:** Missing `include_closed=true` excludes completed deals
- **Silent Fallback:** Any error → returns mock (2 users) without alerting user
- **No Env Loading:** CLICKUP_API_KEY empty on local/Vercel startup

### Test Validation
1. **Unit test:** Mock ClickUp response with 300 tasks → verify pagination fetches all
2. **Integration test:** Call `/api/funil-by-user` with CLICKUP_API_KEY set → verify count matches ClickUp UI ±5%
3. **Manual test:** Create new user in ClickUp, assign lead → verify accordion renders in Funil within 30s
4. **Performance:** Verify no request > 15s; pagination completes in < 5s total
