# Story 1.2: Chart of Accounts API (CRUD)

**Story ID:** 1.2  
**Status:** READY FOR REVIEW  
**Sprint:** Sprint 2  
**Story Points:** 5  
**Priority:** P0

---

## 📋 Story Statement

**As a** accountant  
**I want to** create and manage accounts in the chart of accounts  
**So that** the ERP has a proper account structure for posting entries

---

## ✅ Acceptance Criteria

- [ ] GET /api/accounts - List all accounts with pagination & filtering
- [ ] GET /api/accounts/:id - Retrieve single account details
- [ ] POST /api/accounts - Create new account with validation
  - [ ] Account code must be unique per tenant
  - [ ] Account type must be one of: asset, liability, equity, revenue, expense
  - [ ] Parent account validation (if hierarchical)
- [ ] PATCH /api/accounts/:id - Update account name/parent (non-posted only)
- [ ] DELETE /api/accounts/:id - Soft delete (set is_active = false)
- [ ] RLS enforcement: Each tenant sees only own accounts
- [ ] Error handling: Proper HTTP status codes and messages
- [ ] Input validation: All fields validated before DB insert
