## Acceptance Criteria

1. ✅ **AC1: DRE (Demonstração de Resultado do Exercício) Generation**
   - Endpoint: `GET /api/reports/dre?period=2026-04`
   - Returns: receita_bruta, taxas, receita_liquida, MRR, churn percentage
   - Calculation: SUM of all charges for period with status='paid'
   - Filters: by date range (month, quarter, year)
   - Supports comparison: current vs previous period

2. ✅ **AC2: Cash Flow Statement**
   - Endpoint: `GET /api/reports/cash-flow?start_date=2026-01-01&end_date=2026-12-31`
   - Daily cash flow: inflows (payments received) - outflows (refunds, chargebacks)
   - Shows: projected vs realized
   - Includes: payment method breakdown (PIX, Boleto, CC)
   - Visualizes: cash position trend

3. ✅ **AC3: Payment Status Analysis**
   - Endpoint: `GET /api/reports/payment-status`
   - Returns: pagamentos_em_dia (count + %), pagamentos_pendentes, pagamentos_atrasados
   - Aging analysis: 0-30, 31-60, 61-90, 90+ days past due
   - By customer, by payment method
   - Highlights: at-risk customers

4. ✅ **AC4: Recurring Revenue Metrics**
   - MRR (Monthly Recurring Revenue): SUM active subscriptions
   - ARR (Annual Recurring Revenue): MRR × 12
   - Churn rate: (canceled subscriptions / active at period start) × 100
   - LTV (Lifetime Value): average subscription value × expected lifetime
   - CAC (Customer Acquisition Cost): marketing spend / new customers

5. ✅ **AC5: Forecast & Projections**
   - 30-day forecast: based on next_charge_date from subscriptions
   - 90-day forecast: with confidence scores (high=95%, medium=70%, low=40%)
   - Variance analysis: forecast vs actual (month-over-month)
   - Risk scenarios: if churn increases by 5%, 10%, 15%

6. ✅ **AC6: Export Capabilities**
   - Export formats: PDF, CSV, Excel
   - Scheduled reports: daily/weekly/monthly email to CFO
   - Shareable links: read-only access for stakeholders
   - Historical archive: all reports saved and searchable

7. ✅ **AC7: Data Validation & Integrity**
   - All calculations audited (cross-checked with audit log)
   - Orphaned charges detected (charge without subscription)
   - Duplicate payments flagged
   - Currency validation (all BRL)

8. ✅ **AC8: API Performance**
   - DRE generation: < 2 seconds for 1 year of data
   - Cash flow: < 3 seconds for 2 years of data
   - Proper indexes on charges(status, due_date), subscriptions(next_charge_date)
   - Caching: monthly DRE cached (invalidated daily)

9. ✅ **AC9: Error Handling & Fallbacks**
   - Missing data: defaults to 0 (with log warning)
   - Database disconnection: returns cached reports (< 24h old)
   - Invalid date ranges: returns 400 with helpful message
   - Permission checks: tenants see only own data (RLS)

10. ✅ **AC10: Documentation & Testing**
    - API documentation: endpoint examples, response schemas
    - Test coverage: 80%+ on calculation logic
    - Unit tests: each metric calculation (MRR, churn, LTV, CAC)
    - Integration tests: full report generation end-to-end
    - E2E tests: export in all formats

## Dev Notes

### Calculation Formulas

**DRE (Monthly):**
```
receita_bruta = SUM(charges.amount WHERE status='paid' AND MONTH(paid_at)=current_month)
taxas_cakto = receita_bruta × 0.04 (4% assumed, configurable)
receita_liquida = receita_bruta - taxas_cakto
MRR = SUM(subscriptions.amount WHERE status='active')
churn = COUNT(canceled subscriptions) / COUNT(active at period start) × 100
```

**Cash Flow (Daily):**
```
inflows = SUM(charges.amount WHERE status='paid' AND DATE(paid_at)=target_date)
outflows = SUM(refunds.amount WHERE DATE(refunded_at)=target_date)
net_cash = inflows - outflows
cumulative = previous_day_cumulative + net_cash
```

**Recurring Metrics:**
```
MRR = SUM(plan.amount × COUNT(active subscriptions for plan))
ARR = MRR × 12
LTV = (MRR × expected_lifetime_months) or (average_subscription_value / monthly_churn_rate)
CAC = (marketing_spend / new_customers) [optional - manual input for now]
Churn = (canceled_subscriptions / start_of_period_active) × 100
```

**Forecast (30-day):**
```
forecast_value = SUM(subscriptions.amount WHERE next_charge_date BETWEEN now AND now+30days)
confidence_high = COUNT subscriptions with payment_method='pix' (95% collect rate)
confidence_medium = COUNT with payment_method='boleto' (70% collect rate)
confidence_low = COUNT with payment_method='cc' (40% collect rate)
weighted_forecast = (confidence_high × 0.95) + (confidence_medium × 0.70) + (confidence_low × 0.40)
```

### Key Implementation Details

**Database Optimization:**
```sql
CREATE INDEX idx_charges_status_date ON charges(status, paid_at);
CREATE INDEX idx_charges_due_date ON charges(due_date);
CREATE INDEX idx_subscriptions_status_date ON subscriptions(status, started_at, canceled_at);
CREATE INDEX idx_subscriptions_next_charge ON subscriptions(next_charge_date);
```

**Caching Strategy:**
- DRE for completed months: cached indefinitely (invalidated day 1 of next month)
- Current month DRE: cached 6 hours
- Cash flow: cached 1 hour
- Forecasts: cached 24 hours

**Report Export:**
- PDF: Use pdfkit (Node.js native) — no external service
- CSV: Standard array-to-CSV serialization
- Excel: Use xlsx library (pure JavaScript)
