**Tech Stack:** PostgreSQL, SHA256, NestJS

**Hash Chain Security:**
- Sequential haining: Each hash depends on previous
- Tampering detection: If any record modified, chain breaks
- Immutable logs: RLS prevents modification
- Read-only: Only SELECT allowed for tenants

**Trigger Pattern:**
```sql
CREATE TRIGGER audit_journal_entries
AFTER INSERT/UPDATE/DELETE ON journal_entries
FOR EACH ROW
EXECUTE FUNCTION audit_mutation();

FUNCTION audit_mutation():
  - Capture old/new values
  - Compute hash using previous hash
  - Insert into audit_logs
```

### Implementation summary
Built audit subsystem in three layers:

1. **DB layer (migrations 006 + 007):** `audit_logs` append-only table with composite PK
   `(tenant_id, id BIGSERIAL)`, JSONB old/new, CHAR(64) hash + prev_hash. RLS configured
   for SELECT + INSERT only — UPDATE/DELETE have no policy and FORCE RLS denies them.
   Triggers on the 3 ERP tables (chart_of_accounts, journal_entries, journal_entry_lines)
   call a single `audit_mutation()` function that snapshots the row, looks up the prior
   chain hash for `(tenant_id, table_name, record_id)`, computes the new hash via
   `compute_audit_hash()`, and inserts.

2. **Hash util (`audit-hash.ts`):** Mirrors the SQL formula bit-for-bit. `canonicalJson()`
   sorts object keys recursively to match PG's deterministic JSONB::text. The validator
   walks a chain in id-asc order and reports the exact `brokenAt` id when integrity fails.

3. **API (`audit/`):** Read-only NestJS module with three endpoints (`GET /api/audit`,
   `GET /api/audit/:id`, `GET /api/audit/validate`). Service uses `to_char(... AT TIME
   ZONE UTC)` to project a `changed_at_iso` column whose text format matches the trigger,
   and passes that text verbatim to `computeAuditHash` — avoiding Date round-trip
   timezone-format mismatches.

### Key design decisions
- **Per-record chain key** `(tenant_id, table_name, record_id)`: validates one entity's
  history independently and avoids a global serialization point.
- **`SECURITY DEFINER` on the trigger function:** lets the function insert into
  `audit_logs` even when the calling user has no INSERT grant, while WITH CHECK still
  enforces tenant scoping.
- **Migration renumber 005→006, 006→007:** 005 was already taken by Story 1.4. Story
  file now reflects the actual filenames.
- **No `@nestjs/testing` dependency:** package doesn't ship it; tests instantiate the
  service directly with mocked Pool + Request, matching how the rest of this repo's
  tests are organised.
