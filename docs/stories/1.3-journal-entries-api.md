# Story 1.3: Journal Entries API (Draft → Post → Reverse)

**Story ID:** 1.3  
**Status:** READY FOR REVIEW  
**Sprint:** Sprint 2  
**Story Points:** 13  
**Priority:** P0

---

## 📋 Story Statement

**As an** accountant  
**I want to** create, post, and reverse journal entries with automatic balance validation  
**So that** the ERP enforces double-entry bookkeeping integrity

---

## ✅ Acceptance Criteria

- [ ] POST /api/entries - Create draft entry
  - [ ] Entry created with posted_at = NULL
  - [ ] Lines can be added in request body
  - [ ] Status = DRAFT

- [ ] POST /api/entries/:id/lines - Add lines to entry
  - [ ] Each line is one-sided (debit XOR credit, not both)
  - [ ] Debit and credit are NUMERIC(18,2) >= 0
  - [ ] SUM(debit) = SUM(credit) enforced by trigger

- [ ] POST /api/entries/:id/post - Move entry to posted
  - [ ] Validates balance before posting
  - [ ] Sets posted_at = timestamp, posted_by = current_user
  - [ ] Status = POSTED

- [ ] POST /api/entries/:id/reverse - Create reversal entry
  - [ ] Creates new entry with all lines reversed (debit ↔ credit)
  - [ ] Sets is_reversed = true, reversal_of = original_entry_id
  - [ ] Reversal entry automatically posted

- [ ] GET /api/entries - List entries
  - [ ] Filtering by date range, status (draft/posted)
  - [ ] Pagination support
  - [ ] RLS isolation: tenants see only own entries

- [ ] GET /api/entries/:id/lines - Retrieve all lines for entry
  - [ ] Returns ordered lines (line_order)
  - [ ] Includes account details (code, name)

- [ ] Immutability enforcement:
  - [ ] Posted entries cannot be edited (trigger block_posted_mutation)
  - [ ] Only reversal allowed for posted entries

- [ ] Balance validation:
  - [ ] SUM(debit) = SUM(credit) per entry (trigger enforce_balanced_entry)
  - [ ] One-sided lines only (one_sided_lines trigger)

## 📊 Dev Notes

**Dependencies:** Depends on Story 1.1 (schema) + Story 1.2 (accounts)  
**Assigned to:** @dev  
**Complexity:** HIGH - Core financial logic

**Key Workflows:**

**Draft → Post:**
```
POST /entries → Create with posted_at = NULL
POST /entries/:id/lines → Add lines (one-sided)
POST /entries/:id/post → Validate balance, set posted_at
```

**Reversal:**
```
POST /entries/:id/reverse → Create new entry with reversed lines
→ Auto-post reversal entry
→ Original entry marked is_reversed=true, reversal_of set
```

**Immutability:**
```
Trigger block_posted_mutation() prevents:
- PATCH on posted entry
- DELETE line from posted entry
- Returns 409 Conflict if attempted
```
